# 超级 Jarvis · 个人数据投喂升级方案

> 目标：从「你问它才用工具干活」的助理，升级成**投喂你的工作内容/聊天记录/喜好/行为习惯后，能出主意、辅助决策、甚至预判**的私人超级 Jarvis。
> 原则：**全本地、隐私可控、在现有架构上演进、每阶段可单独验收**。不重写大脑，是给大脑接上「长期记忆 + 主动认知 + 反馈进化」。

---

## 0. 先说结论：现状离目标差什么

你现在的底子其实不差，已有的能用件：

| 已有能力 | 文件 | 现状 |
|---|---|---|
| 工具总线 + 行动闸门 agent 循环 | `leojarvis/agent/loop.py` `gate.py` | ✅ 稳，且已流式化 |
| 记忆库（向量表 + SQLite + 确认队列） | `leojarvis/memory/store.py` `db.py` | ⚠️ 在，但向量是**假哈希** |
| 反思（从事件提炼记忆候选） | `leojarvis/memory/reflect.py` | ⚠️ 在，但只吃近期 events |
| 画像 | `leojarvis/memory/profile.py` + `config/profile.toml` | ⚠️ **静态手填**，不会学 |
| 简报/推送通道 | `leojarvis/briefing/` `notify/hub.py` | ✅ 在，可复用做主动产出 |

**离「超级 Jarvis」的 4 个真实差距：**

1. **记忆是「死文本 + 假向量」**——`embeddings.py` 用 SHA256 哈希冒充语义向量，召回基本靠关键词撞。喂再多数据，它也「记得字、不懂意思」。这是质量天花板。
2. **没有结构化记忆模型**——所有东西都塞进一张 `memories` 表的 `statement` 字段。无法区分「事实 / 发生过的事 / 你的偏好 / 你的行为规律」，也就无法推理。
3. **画像不会学**——`profile.toml` 是你手写的死配置。Jarvis 不会从你的真实行为里更新「他到底在乎什么」。
4. **永远被动**——只有 `/agent/chat` 你问才答。没有「主动出主意 / 预判」的产出回路（虽然简报通道在，但喂的是新闻不是「针对你的建议」）。

**所以升级 = 补这 4 块，而不是从零造。**

## 1. 整体架构（见上方图）

六层，数据从上往下流，反馈从下往上回：

```
① 数据接入层   你投喂的原始素材（连接器拉取）
② 隐私/摄取闸门 脱敏·分块·真向量化·同意分级·来源台账
③ 分层记忆     语义事实 / 情景 / 行为模式 / 人物图谱（确认后转正）
④ 认知引擎     RAG 检索 + 画像风格引擎 + 推理权衡
⑤ 主动产出     出主意 / 辅助决策 / 预判 / 晨间简报
⑥ 反馈学习闭环  你的采纳/否决 → 回流校准 ③④（越用越懂你）
```

第 ⑥ 条回流是「会进化的 Jarvis」和「一个死的知识库」的根本区别。

## 2. 分阶段升级计划（按依赖与价值排序）

每个 Phase 自成闭环、可单独上线、可回滚。建议顺序就是编号顺序——**P1 是地基，不先做后面全是沙上塔**。

### P1 —— 真神经向量 + 分层记忆模型（地基，最高优先）

**为什么先做**：没有真向量 + 结构化记忆，投喂数据只是堆字符串。

**做什么**：
- **真 embedding**：`embeddings.py` 接本地嵌入模型（如 `bge-m3` / `paraphrase-multilingual-MiniLM`，CPU 可跑、离线、中英都行），保留哈希向量做导入失败兜底。一次性重建索引脚本。
- **分层记忆 schema**：`memories` 表扩展 `layer` 字段（或拆表）：
  - `fact`（语义事实：人、项目、持仓、长期偏好）
  - `episode`（情景：某天发生了什么、某次对话）
  - `pattern`（行为规律：「他周一早上先看持仓」「他拒绝周末打扰」）
  - `entity`（人物/项目图谱节点 + 关系）
- 复用现有 `confidence / salience / decay_after / status(pending→active)` 字段——确认队列机制保留，**所有从你数据里抽出的记忆先进 pending，你点头才转正**。

**验收**：构造若干 query，真向量 top-5 命中明显优于哈希；新记忆按 layer 落库；pending→active 流程通。

---

### P2 —— 数据接入连接器 + 隐私闸门（让数据「灌得进、灌得安全」）

**为什么**：超级 Jarvis 的燃料是你的真实数据。这一层决定能喂什么、喂得安不安全。

**做什么**：
- **连接器**（逐个加，复用现有 `ingest/` 框架）：
  - 工作内容：本地文档/代码目录、会议纪要、Apple Notes
  - 聊天记录：导出文件（微信/Telegram 等）批量导入 + 脱敏
  - 个人喜好：一次性结构化问卷（喜欢什么、讨厌什么、不想被打扰的时段……）→ 直接写 `fact` 层
  - 行为习惯：日历、系统使用、浏览历史（已有 `browser_history.py`）、agent 操作日志
- **隐私闸门**（不可省，这是个人数据系统的命门）：
  - 全程本地：嵌入、存储、推理都不出本机（LLM 调用是唯一外发，需明确告知）
  - **同意分级**：每类数据可单独开关；敏感项（密码、财务明细、私密对话）默认拒喂
  - **脱敏**：复用 `personal_notes.py` 已有的 `_SENSITIVE_PATTERNS`，导入前过滤
  - **来源台账 + 可删除**：每条记忆记住「来自哪个文件/哪条消息」，可一键溯源、可一键删除该来源的全部衍生记忆（被遗忘权）

**验收**：导入一批真实文档/聊天 → 脱敏生效 → 记忆带来源 → 能按来源删除；敏感项被拦。

---

### P3 —— 反思升级：从数据里「学画像、抽规律」（让它真的懂你）

**为什么**：光存不够，要从原始数据里**提炼出关于你的判断**。

**做什么**：
- `reflect.py` 升级：不只吃近期 events，定期对**全量个人数据**做批量提炼（批量 LLM，省成本）：
  - 抽 `fact`：「Leo 在做 LeoJarvis 项目」「持仓里有 NVDA」
  - 抽 `pattern`：「常在深夜写代码」「对周末通知反感」「决策前喜欢看数据对比」
  - 更新 `entity` 图谱：谁是谁、什么项目关联什么人
- **动态画像**：`profile.py` 从「读死 toml」变成「toml 种子 + 学习到的 fact/pattern 合成」。画像随你的真实行为漂移。
- 全部走**确认队列**：抽出的画像更新先 pending，你在「长期记忆」视图确认。

**验收**：喂一段时间数据后，画像里出现你**没手填、但被正确推断**的条目；规律抽取人工抽检准确。

---

### P4 —— 主动认知：出主意 / 辅助决策 / 预判（核心质变）

**为什么**：这是你要的「超级」——从应答变主动。

**做什么**（复用现有 agent 循环 + 简报 + 推送通道）：
- **RAG 全面接管**：agent 每次回答前，先按 layer 加权检索相关记忆（事实+情景+偏好）喂进 prompt → 回答带「你的上下文」。
- **风格引擎**：用你的聊天记录/写作学你的口吻，产出读起来像「你的参谋」而不是通用 AI。
- **三种主动产出**：
  - **出主意**：基于情境主动给建议（「这个项目你上次卡在 X，这次可以先 Y」）
  - **辅助决策**：给选项 + 利弊 + 结合你偏好的推荐 + 打分（决策卡片）
  - **预判**：识别模式提前提醒（「按你习惯今天该交周报了」「这支票到了你设的关注价」）
- **晨间主动简报升级**：现有简报塞「针对你的待办/建议/风险」，不只是新闻。
- ⚠️ **安全护栏**：决策类输出只「建议」，真正动钱/发消息/删数据仍走**行动闸门确认**（你现有的 gate 机制）。不替你拍板不可逆的事。

**验收**：同样问题，升级后回答明显带你的个人上下文；能主动推一条「针对你」的建议/预判；高风险动作仍需确认。

---

### P5 —— 反馈进化闭环（让它越用越准）

**为什么**：没有反馈，画像和记忆会越来越偏，最后变成「自信地说错」。

**做什么**（复用现有 `/feedback` + 确认机制）：
- 每条主动产出带「采纳 / 忽略 / 纠正」按钮（iOS + Web 都加）。
- 反馈回流：
  - 采纳 → 强化相关记忆的 `salience/confidence`
  - 忽略 → 衰减
  - 纠正 → 直接修记忆 + 记一条新 fact
- 定期「记忆体检」：低置信、长期未命中、被多次否决的记忆自动降权/归档（接上你已做的数据保留/VACUUM）。

**验收**：连续用一阵，被否决的建议类型不再反复出现；采纳过的方向权重上升、命中更准。

## 3. 关键技术选型与红线

**选型（稳健优先，全可本地）**
- 嵌入：本地 `bge-m3`/MiniLM 级模型（CPU 可跑）。**这是 P1 必须接受的一个新依赖**——是「真懂」与「假懂」的分界。
- 向量库：继续用现有 LanceDB（已在 `store.py`），只换 embedding 来源。
- 推理 LLM：现有 DeepSeek（端点无关协议，随时可换更强模型）。批量提炼用批处理省钱（你已做过 judge_batch，同套路）。

**红线（个人数据系统不可破）**
1. **本地优先**：除 LLM 调用外不外发；LLM 外发的内容要可审计、敏感项不进 prompt。
2. **同意分级 + 可遗忘**：每类数据可拒、每条记忆可溯源可删。
3. **建议 ≠ 执行**：决策/预判只产出建议，动钱/对外/删除一律走行动闸门确认。
4. **可解释**：每条建议能说清「基于你的哪条记忆/习惯」，不做黑箱。
5. **会进化但不漂移**：靠反馈闭环 + 记忆体检防止「自信地记错」。

## 4. 路线图一览

| 阶段 | 一句话 | 产出 | 依赖 |
|---|---|---|---|
| **P1** | 真向量 + 分层记忆 | 记得住、分得清、能检索 | 无（地基） |
| **P2** | 数据连接器 + 隐私闸门 | 灌得进、灌得安全 | P1 |
| **P3** | 反思学画像抽规律 | 真的懂你是谁 | P1·P2 |
| **P4** | 出主意/决策/预判 | 主动的超级 Jarvis | P1·P3 |
| **P5** | 反馈进化闭环 | 越用越准 | P4 |

**建议**：先 P1 打地基（不接受新嵌入依赖就退而求其次用 BM25，但召回质量留在天花板下，不推荐）。P1 通了，P2/P3 可并行喂数据，P4 才是你最想要的体感质变，P5 让它长期不退化。

## 4.5 实现状态（P1–P5 已全部落地 ✅）

本方案已实现并测试（138 后端测试全过，零回归）。默认全本地、可降级、可回滚。

| 阶段 | 状态 | 关键产物 |
|---|---|---|
| **P1** 真向量 + 分层记忆 | ✅ | `embeddings.py` 多后端可降级（http / sentence-transformers / hash）；`memories` 表加 `layer/origin/source_ref`；`store.py` 分层检索 + `forget_vectors`；`db` 加 `list_memories_by_layer / adjust_memory / delete_memories_by_source / memory_layer_counts / memory_health_sweep` |
| **P2** 连接器 + 隐私闸门 | ✅ | `personal_data/` 模块：4 类连接器（work/chat/preference/behavior）+ 同意分级 + 红线词 + 脱敏 + 来源台账 + 被遗忘权。API：`/personal-data/status`、`/ingest/{text,chat,preferences,behavior}`、`/forget` |
| **P3** 反思学画像 + 抽规律 | ✅ | `reflect.reflect_personal_data()` 把情景记忆提炼成 fact/pattern/entity（走确认队列）；`profile.py` 动态画像（toml 种子 + 已学事实/规律）。API：`/memory/distill` |
| **P4** 出主意/决策/预判 | ✅ | `cognition.py`：`advise/decide/anticipate`；agent 每轮带「关于你」的 fact/pattern 上下文（RAG）。API：`/cognition/{advise,decide,anticipate}`。**只产建议，执行仍走行动闸门** |
| **P5** 反馈进化 + 记忆体检 | ✅ | `/cognition/feedback`（accept 强化 / ignore 衰减 / correct 改写）；`db.memory_health_sweep` 接入每日维护，低置信/久未更新归档。`/metrics` 新增记忆分层计数 + 是否真向量 |

**怎么开始用（最小闭环）：**
1. （可选但推荐）启用真向量：在 `config/settings.toml [embeddings]` 设 `base_url` 指向本地 Ollama/embedding 服务，或 `pip install sentence-transformers`。不配也能跑（哈希兜底）。
2. 投喂：`POST /personal-data/ingest/preferences`（喜好问卷，本人填，直接生效）+ `/ingest/text`（工作文档）。聊天默认关，要在 `user_settings.personal_data.consent.chat` 打开。
3. 提炼：`POST /memory/distill` → 在「长期记忆」视图确认 fact/pattern。
4. 用起来：`/cognition/advise`、`/decide`、`/anticipate`；日常对话已自动带个人上下文。
5. 反馈：对建议点 accept/ignore/correct（`/cognition/feedback`），越用越准。

**隐私红线（已写死在代码里）：** 全本地（仅 LLM 外发，敏感项脱敏后才进 prompt）；同意分级 + 红线词拦截 + 按来源一键遗忘；主动产出只建议不执行；记忆体检防漂移。

## 5. 你需要现在拍的板（影响方案落地）
